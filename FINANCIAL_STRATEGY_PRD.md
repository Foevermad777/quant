# 第一阶段 MVP：金融策略与风控需求文档 (PRD)

版本：0.1
日期：2026-07-04
定位：本文件是《量化交易系统项目协作章程》（`PROJECT_CHARTER.md`）的金融业务补充包。工程架构、分工与 JSON 信号结构以章程为准，本文件仅统摄 **金融逻辑边界、大模型风控纪律** 以及 **MVP标的池定义**。

## 1. 核心约束：金融三大机械纪律
在调用 LLM 进行新闻解读、公告分析和生成 `reasoning_summary` 时，Prompt 必须强制注入以下纪律，如有违反，风控层应予拦截：

1. **数据溯源机械纪律**
   - 引用任何财务报表、宏观经济指标（CPI、非农等）或突发新闻时，必须强制提取精确发布日期。
   - 必须严格区分财务口径（自然年 CY vs 财年 FY）。
   - 【禁止】“摘要式”无源结论。
2. **市场判断机械纪律**
   - 拒绝模糊术语。禁止使用类似“美债跌”、“放水”等口语，必须规范化为“美债收益率下降/美债价格上涨”、“美元流动性扩张”等精确词汇。
   - 【强制】每个看多/看空判断，必须在 `invalid_conditions`（失效条件）数组中写明反证情景。
3. **多情景与置信度纪律**
   - 抛弃绝对化/情绪化词汇（如“暴跌”、“绝佳底仓”）。
   - 报告生成模块必须要求大模型按 Base/Bull/Bear（基准/乐观/悲观）输出条件概率推演。
   - `confidence` 的得分必须附带逻辑支撑，且在极端单边情绪下自动降权。

## 2. 标的池扩展路线图 (Target Pool Roadmap)
为避免 Phase 1 阶段处理全市场数据导致 API 触发风控或 LLM 上下文超载，系统采取“先熟悉后扩张、多市场全覆盖”的阶梯式验证路线（由 CEO 制定）：

*   **Phase 1 MVP（起步验证期）：A股核心资产**
    *   标的：用户高辨识度的自选股或沪深300核心成分股。
    *   目的：在 CEO 最熟悉的市场建立直觉反馈，快速跑通闭环，重点验证系统对政策底、信贷周期、主力资金流向的“排雷”有效性。
*   **Phase 2 扩张（宏观敏感验证）：美股核心科技**
    *   标的：纳斯达克核心七巨头 (Magnificent 7) + 费城半导体头部标的。
    *   目的：系统框架稳定后，验证其对美联储利率决议、非农就业、全球巨头资本开支(CapEx)等多情景宏观推演的准确率。
*   **Phase 3 终极（全天候流动性验证）：加密核心资产 (Crypto)**
    *   标的：BTC, ETH, SOL 等高流动性代币。
    *   目的：脱离传统财报束缚，验证系统对 7x24 小时新闻情绪、资金费率、链上大额异动的极速响应与量化指标判定能力。
