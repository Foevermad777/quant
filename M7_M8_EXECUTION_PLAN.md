## 2. M7：信号→模拟盘执行器（预计 2-3 天）

### 2.1 目录与环境

```
量化系统/
├── executor/            # 本次新建的 Python 包（我们项目的第一份自研代码）
│   ├── __init__.py
│   ├── config.py        # 上表参数集中于此，全部可配
│   ├── signal_reader.py # 只读 DSA sqlite：decision_signals + analysis_history（S1 口径对账）
│   ├── rules.py         # A股规则组件：T+1 记账、涨跌停判定、停牌判定
│   ├── models.py        # 可插拔小类：SlippageModel / FeeModel / FillModel
│   ├── ledger.py        # 独立台账（runtime_data/quant/paper.db，自有 SQLite）
│   ├── engine.py        # 主循环：昨日信号成交 + 持仓逐日盯市
│   └── tests/           # 规则单测（见 2.4）
├── ops/run_executor_daily.sh   # 包装脚本（模式同 run_dsa_daily.sh）
```

设计决策说明（供评审）：**台账自建而不写 DSA portfolio**。理由：① 红线上更干净（对 DSA 零写入）；② 免去常驻 DSA server 的运维负担；③ 台账 schema 属于"我们的核心资产"，必须可整体搬迁（章程第 9 节）。DSA portfolio 镜像留作后续可选项（仅为了 Web 界面可视化，不承担记账职责）。

### 2.2 A 股规则实现（抄作业清单，来自 QUANT_OSS_SURVEY 第三节）

1. **T+1**：抄 RQAlpha `position.py` 的 `_old_quantity/closable` 记账法——持仓分"今仓/老仓"，每日收盘结算时 `_old_quantity = _quantity`，可卖量 = 老仓。禁止用"记录买入日期"的实现。
2. **涨跌停**：抄 RQAlpha `matcher.py` 语义——开盘价等于涨停价则买单不成交（记 `unfilled_limit_up`），等于跌停价则卖单不成交（顺延次日再试直至信号过期）；做成配置开关。涨跌停价按前收盘 ±10%（ST ±5%，创业板 300 开头 ±20%）计算并四舍五入到分。
3. **停牌**：当日 `stock_daily` 无该股 bar 即视为停牌，跳过并顺延。
4. **止损/止盈部件化**：抄 Hikyuu——StopLoss/TakeProfit/PositionSizing 各为独立组件注入 engine，不揉进主循环。

### 2.3 主循环时序（每交易日 18:40，接在 DSA 17:58 跑批之后）

前提核验（Codex 实现前先确认）：DSA 当晚跑批会把**当日完整日线**写入 stock_daily——抽一天验证后在验收记录注明。若不成立，改为次日 09:00 运行并顺延一天，方案 B 写在代码注释里。

```
1. 熔断检查：DSA 当日跑批是否成功（当日 analysis_history 有记录）；失败则本轮只做持仓盯市不开新仓
2. 读昨日及更早的 active 信号（排除今日新产信号），按 S1 对账 + S5 同股取最新去重
3. 开仓：对合格 buy/add 信号按**限价单语义**成交（见第 1 节成交假设：开盘 ≤ entry_high 按开盘价；日内触及按 entry_high；未触及顺延），过涨跌停/停牌/资金检查，写台账；当日买入且同日 low 破止损价 → 记止损触发，T+1 次日开盘价执行卖出（保守口径）
4. 盯市：对全部持仓，用今日 bar 判止损/止盈触发（歧义止损优先）、处理 sell 信号平仓、过期处理
5. 结算：T+1 老仓同步、组合快照（现金/市值/浮盈）写台账
6. 幂等保证：以 (signal_id, trade_date) 唯一键防重复成交，重跑当日安全
7. 留证：每日执行日志 runtime_data/quant/executor_YYYYMMDD.log
```

**回填模式**：首次上线时对 2026-07-06 起已积累的信号按历史日线补执行，让模拟盘与 Day 1 对齐（一次性命令 `python -m executor.engine --backfill-from 2026-07-06`）。

## 3. M8：周复盘聚合（预计 1 天，07-18 前完成，供 07-21 复盘会用）

`ops/weekly_review.py`，输出 `runtime_data/quant/WEEKLY_REVIEW_YYYYMMDD.md`：

1. **信号面**（读 DSA outcomes + decision_signals）：信号数、方向胜率（T+1/3/5/10 各档）、按 action/置信度分组的胜率（为校准回路攒数据）、口径冲突与数据缺口清单；
2. **模拟盘面**（读自有台账）：已实现/浮动盈亏、**盈亏比、期望值、最大回撤**（章程第 7 节要求而 DSA 缺失的三个指标）、止损/止盈触发率、平均持仓天数；
3. **双基线对比**（抄 AgenticTrading）：同期沪深300 指数收益、等权买入持有 5 只股收益——我们的一切成绩必须相对这两条基线报告；
4. **统计诚实性**（抄 PyBroker 思路，numpy 自实现即可）：对模拟盘收益做 bootstrap 重采样，报告 90% 置信区间——样本小的时候，置信区间比单点数字诚实；
5. **延迟回填反思**（抄 TradingAgents，v1 简化版）：每条已了结信号回填已实现收益与对沪深300 的 alpha。

验收标准：
- [ ] 用 14 天真实数据能一键生成完整报告；
- [ ] 盈亏比/期望值/最大回撤三个指标我用独立 SQL 复算一致；
- [ ] 双基线数据来源与区间对齐无偏移（基线也按同样的成交假设计费）。
