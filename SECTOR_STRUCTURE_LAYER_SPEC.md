# 板块结构 / 产业链—拥挤度层（Phase 2 候选模块）规格草稿

版本：0.2（草稿，未排期）｜ 日期：2026-07-13 ｜ 制定：Claude ｜ 状态：**候选，待 CEO 决定是否进 Phase 2**
v0.2 变更：按 CEO 澄清，luopan 从"纯人肉助手"重定位为**漏斗顶端的主题假设生成器**（经留痕、证伪、人工确认后进 watchlist），确立三段漏斗；选池方式定为 **A（人在环内）**。

## 0. 一句话

宏观选赛道 → 微观选时机的**三段漏斗**：luopan 检索产业链结构提出"往哪看"的**假设** → 量化层用数据**证伪并过滤** → DSA + 护栏定"何时买"。luopan 决定在哪片水域下网，**从不决定这条鱼买不买、何时买**。

## 1. 核心原则（先划红线）

- **luopan 是假设生成器，不是结论**：它检索的是已公开的共识，强在"画结构（谁上游、谁有定价权）"，弱在"抢时间（谁还没涨）"。故其产出一律当**待证伪假设**，由数据放行或枪毙。这既实现 CEO 要的"有针对性只看某产业某环节、不复盘全盘"，又守住"LLM 叙事不进买入决策"的底线——旧闻只当搜索种子，绝不当买入理由。
- **三段职责不可混**：① 找方向（luopan）｜② 证伪+选池（量化）｜③ 定时机（DSA+G4/G5）。任一段的产出不得越权替代下一段。
- **好公司 ≠ 好价格 ≠ 好时机**：高话语权/高毛利是"质量"属性，不等于现在该买（luopan 自身方法论亦如此，见其英伟达案例：质量过关但价格未过安全边际）。质量、估值/拥挤度、择时是三道独立闸门。
- **继承既有纪律**：不改 DSA 源码只读其库；任何影响 paper 交易的用法先过 OOS 红线 + 防未来函数护栏（对齐 G3）；物理隔离（独立模块/独立表/零共享运行期代码，沿用美股 fork 纪律）。

## 2. 三段漏斗与数据流

```
① 主题假设层(luopan, 低频)      ② 量化证伪+选池层(本模块核心)          ③ 择时层(现有)
产业链结构/定价权检索      →   留痕→lead-lag证伪→拥挤度/流动性/       →  DSA + G4/G5 + 执行器
产出: 候选方向+候选环节         定价权过滤 → 候选清单 →【CEO 点头】       定买卖点(不进 LLM 叙事)
      (写 theme_hypotheses)      → 进 watchlist(写 sector_context)
```

- 新模块落 `ops/sector_structure.py` + `executor/sector/`，companion 库新表 `theme_hypotheses`（① 的留痕）+ `sector_context`（②的产出）。均不写 DSA 库、不进 decision_signals。

## 3. 第①段：主题假设层（luopan）——三条纪律

luopan（`.claude/skills/luopan/`）低频运行（如按主题/周期触发，不是每日全市场跑），产出候选产业链方向 + 各环节定价权画像。三条强制纪律：

1. **提名全留痕（防选择性记忆）**：每次 luopan 提的方向/环节/理由，带 UTC 时间戳写 `theme_hypotheses`。禁止只记蒙对的那次。
2. **选池层也要 OOS 命中率**：事后量化——它提名的环节，在提名日之后是否真跑赢基准？把 OOS 红线从信号层延伸到**选池层**；命中率证不出 edge，本段只能停在"给人参考"，不得进 watchlist 自动化。
3. **量化反向筛 luopan（防操纵/防已晚）**：luopan 给主题后，必过**拥挤度 + 流动性 + 定价权**三过滤器。luopan 偏爱的"隐性受益者"上下游常是易操纵的小盘、低流动性股（如当下人形机器人产业链概念股），结构诱人不等于可交易；数据剔掉"质好但晚/不可交易"的名字。

## 4. 第②段：量化证伪 + 选池层（本模块核心）

### 4.1 数据源（复用现有 provider，只读）
申万/中信行业分类 + 板块指数（akshare/tushare）；估值分位、成交占比、板块宽度、（可选）资金流。**产业链连接是最弱一环**：v0.1 用人工维护静态链图 + 数据实测 lead-lag，不假装有真实投入产出表。

### 4.2 因子
- **板块动量（截面）**：多窗口板块收益排名/分位，仅用 ≤T 数据。
- **产业链 lead-lag**：静态链图相邻上下游对算滞后互相关，**用数据估传导时滞**；上游已动、下游按历史时滞未反应 → 标"潜在传导窗口"。时滞可回测，不拍脑袋。
- **拥挤度因子（把"追涨危险"量化）**：估值分位+成交占比+宽度+(可选)资金流加速度的 z 分合成；**用途是降权/否决，不是生成**，直接编码"大家都知道它火=你入场是追涨"。
- **议价能力过滤**：板块级毛利率水平/趋势，压低"位置靠上但无定价权的大宗"环节，落地 luopan"权力按议价能力非产业链位置"洞察。
- **流动性/可交易过滤**：成交额、换手、盘子大小，剔除易操纵小盘。

### 4.3 产出
`sector_context` 表 + 候选清单（含每个环节：结构成立度 / lead-lag 时滞 / 拥挤度 / 定价权 / 流动性标记）。**A 模式：候选清单需 CEO 点头才进 watchlist。**

## 5. 第③段：择时层（现有，不变）

进了 watchlist 的股，走现有 DSA + G4/G5 + 执行器定买卖点。**此段坚决不进 LLM 叙事**，纪律不变。DSA 的严进策略（乖离率/均线多头/缩量回踩）是微观择时，**不替代第②段的估值/拥挤度闸门**。

## 6. OOS 验证

- **在板块/环节层验证，不在 5 股池验证**（样本太小）。样本内估 lead-lag 时滞与拥挤度阈值 → 冻结 → 后段留出窗口 OOS（对齐红队 OOS 红线）。
- 严格 point-in-time：基本面/估值用披露滞后后的数据，防未来函数（对齐 G3）。
- 基准 = 不加本层的板块等权 / 现有 baseline；比风险调整后收益、回撤、**追涨情形避损**、**选池命中率**。
- 必须诚实报告失败模式：拥挤度阈值 regime 依赖、静态链图错配、时滞不稳定、选池命中率不显著。任一未过 → 停在顾问态，不进护栏/自动化。

## 7. 与执行器解耦（两态，都 opt-in、都门控）

- **2a 顾问态（先做）**：只写 `theme_hypotheses` + `sector_context` + dashboard 展示，供 CEO 选池参考。执行器完全不读，零管道耦合。
- **2b 护栏态（后做，须先过 §6）**：执行器可选查板块拥挤度，对入场降仓位/否决——是护栏不是信号源，**永不自产买入**（沿用"以护栏后 advice 为准、冲突则跳过"的 S1 口径）。挂 flag、默认关、过 G4 式门控 + OOS 证明后才可影响 paper。

## 8. 与 OpenClaw"合流"提案的差异（存档对齐，防三方各执一词）

OpenClaw 提出"宏观选赛道→微观选时机"闭环，大方向与本规格一致，但其表述有三处按字面执行会绕过护栏，已按 CEO 决定拉回：

1. OpenClaw"罗盘**自动**生成/导入自选股" = 本规格否掉的 **B（全自动）**。本规格定 **A（人在环内）**：LLM 提名 + 量化证伪 → 候选清单 → CEO 点头 → 才进 watchlist。
2. OpenClaw 链路"罗盘选公司→直接进自选股→DSA 择时"**压掉了第②段**（量化证伪+拥挤度/流动性过滤），等于把 luopan 叙事升格为选股结论。本规格坚持完整三段。
3. OpenClaw 举例（鸣志/柯力/恒立等人形机器人产业链概念股）正是"结构诱人但可能已拥挤/易操纵"的典型，是第②段过滤器**必需**的证据，而非可直接买入的清单。其是否为真 edge，由数据裁决，不由叙事裁决。

## 9. 分期与风险

- **M-Sector-0**：静态链图 v0.1 + 数据源接通 + `theme_hypotheses`/`sector_context` 建表（顾问态可展示）。
- **M-Sector-1**：三类因子 + 板块层 OOS 回测 + **选池命中率**报告（红线内）。
- **M-Sector-2**（仅当 M1 过 OOS）：护栏态接入执行器，flag 默认关，先影子后灰度。
- 风险：产业链连接数据弱（§4.1 标注）；拥挤度/时滞 regime 依赖；范围蔓延（本层只做"选池+护栏"，不扩成大择时/多因子工程，升级需 CEO 批）。
