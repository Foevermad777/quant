# 板块结构 / 产业链—拥挤度层（Phase 2 候选模块）规格草稿

版本：0.1（草稿，未排期）｜ 日期：2026-07-13 ｜ 制定：Claude ｜ 状态：**候选，待 CEO 决定是否进 Phase 2**

## 0. 一句话

不再只看 5+5 个股，而是先看**整个市场的板块结构**：哪个板块在动、产业链上下游谁领先谁滞后、以及"大家都知道它火"的**拥挤度**——把"追涨很危险"这句直觉变成**可计算、可回测、可解耦**的量化层，作为选池与仓位护栏的输入，而不是又一个 LLM 叙事信号。

## 1. 动机与边界（先划清红线）

- **动机**：单看几只股 = 影响因子单一、易踩追涨。板块轮动 + 产业链 lead-lag + 拥挤度是成熟量化概念（对应 [QUANT_OSS_SURVEY.md](QUANT_OSS_SURVEY.md) 的 Phase 2 方向）。
- **与 luopan 的分工**：luopan（已装进 `.claude/skills/luopan/`）是**给人看**的定性研报助手，用于**选池阶段人肉研究**，**永不接入执行器**。本层是**给系统用**的定量层，产出的是数字（动量分位、lead-lag、拥挤度 z 分），不是 LLM 判断文字。二者不混。
- **红线（继承既有纪律）**：① 本层**不自己生成买入信号**，只产板块上下文 + 拥挤度护栏；② 任何影响 paper 交易的用法必须先过 OOS 红线（见 §4）与防未来函数护栏（对齐 G3）；③ 不改 DSA 源码，只读其库；④ luopan 的核心提醒——**上下游按"议价能力"而非产业链物理位置分**——写死进因子（见 §3.4），防"买没有定价权的上游大宗"陷阱。

## 3. 数据源与因子定义

### 3.1 数据源（全部复用现有 provider，只读）
- **行业分类**：申万一级/二级（akshare `sw_index`）或中信；个股→板块映射每日快照留痕。
- **板块指数行情**：akshare/tushare 行业指数日线（复用现有代理与 NO_PROXY 直连纪律）。
- **拥挤度原料**：板块 PE/PB 历史分位、成交额占全市场比、板块宽度（成分股在 MA20/60 上占比）、（可选）板块资金流。
- **产业链连接**：**本层最弱的一环，如实标注**。v0.1 用**人工维护的静态产业链图**（如 AI→算力→金属→能源，一次性 curate + 版本化）+ 数据实测的 lead-lag 验证；不假装有真实投入产出表。v0.2 再考虑营收暴露/概念板块关系补强。

### 3.2 板块动量（截面）
- 各板块指数多窗口收益（如 20/60/120 日），截面排名/分位。仅用 ≤T 数据。

### 3.3 产业链 lead-lag（传导）
- 对静态链图里相邻上下游板块对，算滞后互相关，**用数据估计传导时滞**；当上游已动、下游按历史时滞尚未反应时，标"下游潜在传导窗口"。**时滞是估出来的、可回测的，不是拍脑袋。**

### 3.4 拥挤度因子（核心，把"追涨危险"量化）
- 拥挤度 = z 分合成：估值分位（越高越拥挤）+ 成交占比（放量拥挤）+ 宽度（过热）+（可选）资金流加速度。
- **用途是"阻尼/否决"而非"生成"**：高拥挤板块 → 对该板块的动量入场**降权或否决**，直接编码"大家都知道它火 = 你入场是追涨"。

### 3.5 议价能力过滤（luopan 洞察落地）
- 板块级毛利率水平与趋势作为过滤：**优先有定价权（毛利率高/升）的环节**，压低"位置靠上但无定价权的大宗商品"环节权重。防止机械"买上游"。

## 4. OOS 验证（怎么证明它有用，而不是过拟合）

- **在板块层验证，不在 5 股池验证**：验证池只有 5 只，样本太小。先证"板块轮动+拥挤度在**板块指数**上有 OOS 预测力"，与 5 股池解耦。
- **协议**：① 样本内窗口估 lead-lag 时滞与拥挤度阈值 → **冻结** → 后段留出窗口做 OOS（对齐红队"OOS 刻入红线"）。② 严格 point-in-time：基本面/估值用**披露滞后后**的数据，防未来函数（对齐 G3 护栏）。③ 基准 = 不加本层的板块等权/现有 5+5 baseline；比较风险调整后收益、最大回撤、追涨情形下的避损。
- **必须诚实报告的失败模式**：拥挤度阈值的 regime 依赖、静态链图错配、样本外失效、时滞不稳定。任一未过 → 只能停在 §2a 顾问态，不得进 §2b 护栏态。

## 5. 与执行器解耦（两态，都 opt-in、都门控）

- **2a 顾问态（先做，零风险）**：只写 `sector_context` + dashboard 展示，供**人**在选池/调仓时参考。执行器完全不读。可立即验证因子而不碰 paper 交易。
- **2b 护栏态（后做，需先过 §4）**：执行器**可选**查询板块拥挤度，对入场做**降仓位/否决**——是护栏不是信号源，**永不自己产买入**（沿用"以护栏后 advice 为准、冲突则跳过"的 S1 口径）。挂在 flag 后、默认关、过 G4 式门控 + OOS 证明后才可影响 paper。
- **物理隔离**：独立模块 + 独立表 + 只读 DSA，与核心执行器零共享运行期代码；bug 不互串。

## 6. 分期与验收

- **M-Sector-0**：静态产业链图 v0.1 + 数据源接通 + `sector_context` 建表（顾问态可展示）。
- **M-Sector-1**：三类因子实现 + 板块层 OOS 回测报告（红线内），产出"是否有预测力"的定性结论。
- **M-Sector-2**（仅当 M1 过 OOS）：护栏态接入执行器，flag 默认关，先影子运行再灰度。
- 每阶段按项目既有验收：真实数据跑、防未来函数自检、OOS 红线、与执行器解耦硬证据。
