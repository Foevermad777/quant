# 开源量化项目调研与借鉴地图

版本：0.1
日期：2026-07-06
调研方式：OpenClaw 人工清单（5 项）+ Claude 三路并行检索（A 股框架补漏 / LLM×量化同构项目 / 论断核实与实盘通道摸底），所有 GitHub 数据为 2026-07-06 当日查证快照，非记忆。
用途：回答"我们的外挂量化层是否要参考成熟开源项目"，并为 M7 执行器、Phase 2 选型、Phase 4 实盘预研提供依据。

## 一、总原则（本次调研的最终答案）

**要参考，但"抄设计、不抄依赖"。** 全网不存在"消费外部 LLM 信号 → A 股规则感知的模拟盘执行"的现成项目——所有 A 股向 LLM 交易项目（TradingAgents-CN、A_Share_investment_Agent 等）**全都绕开了 T+1/涨跌停/停牌的成交真实性问题**。这个共同盲区正是我们自研执行器的稀缺价值所在；而每个自研件该抄谁的作业，本文第三节给出到文件级的答案。

另一个重要验证：TradingAgents（2024-12 创建，18 个月 91k stars）的闭环与我们架构完全同构——"LLM 研判 → Pydantic 结构化 → SQLite 落库 → 下次运行用已实现收益回填 pending 决策 + 生成反思"。**社区已经收敛到和我们相同的形态**，说明方向选对了；我们不需要发明任何复盘指标，直接采用公认模式即可。

## 二、OpenClaw 清单核实结论：5 项中 4 项属实，1 项需实质修正

| 项目 | OpenClaw 判断 | 核实结论（2026-07-06） |
|---|---|---|
| vn.py | 高推荐，A股/期货实盘首选 | **需修正**：项目活跃属实（42.7k stars，4.4.0），但"A股实盘首选"隐含机构视角——官方股票 gateway（XTP/华鑫奇点/东财EMT）门槛 50 万~1000 万，且官方对散户主流 QMT 通道**只有数据接口没有下单 gateway**。散户现实路径是 miniQMT+xtquant（10-50 万门槛），vn.py 非必经之路 |
| QUANTAXIS | 中推荐，臃肿维护存疑 | 属实且偏乐观：近 6 年无 stable release（停在 2020-12），alpha 常态化、单人维护。不作依赖，但其 T+1 可卖持仓分离记账、涨跌停不可成交的实现值得抄 |
| Freqtrade | 高推荐，仅 crypto | 完全属实（52.1k stars，月度发版）。虽与 A 股零交集，但其 **dry-run 与 live 共用同一执行代码路径**的设计原则、护栏体系（每日亏损熔断/冷却期）是我们执行器的优质参考 |
| LEAN | 高推荐，A股适配差 | 完全属实。排除正确，但其 **Reality Modeling 分层**（Fee/Slippage/Fill/Settlement 各自独立可插拔）是教科书设计，执行器的成交假设应照此拆成可替换小类 |
| Qlib | 中推荐，投研强离下单远 | 属实且**推荐度应上调**：45.7k stars 远超同类，官方已集成 RD-Agent（LLM 自动因子研发）。Phase 2 若走 ML 因子路线，Qlib+RD-Agent 是第一候选。注意官方 A 股数据集因数据安全政策暂禁用，需自灌数据（我们有 DSA 数据链，无碍） |

## 三、按组件的借鉴地图（M7 执行器直接引用本节）

### 执行器（本周自研，抄四家设计）

1. **A 股撮合规则 → 抄 RQAlpha**（全场最干净，已核实到文件）：
   - 涨跌停判定：`rqalpha_mod_sys_simulation/matcher.py`——涨停禁买/跌停禁卖做成配置开关而非硬编码；
   - **T+1 记账法：`position.py` 的 `_old_quantity/closable` 机制**——当日买入量 = `_quantity - _old_quantity` 不可平，每日结算时同步，比"记录买入日期"干净得多，直接采用；
   - `signal_broker.py`（信号模式：不撮合直接按信号价成交）与我们场景一比一对应，整体读一遍。
   - 注：matcher 耦合其框架无法单独 import，**抄逻辑不引框架**——抄参考实现不算"从零写撮合"，不踩红线。
2. **止损/止盈/仓位部件化 → 抄 Hikyuu**：ST/TP/MM 各自独立成策略对象注入交易管理器，将来"固定百分比止损"换"ATR 止损"只改一个部件。
3. **成交假设可插拔 → 抄 LEAN**：滑点/费用/成交模型拆成可替换小类，不写死在执行循环里。
4. **模拟与实盘同路径 → 抄 Freqtrade**：执行器接口从第一天就设计成"dry-run 和 live 走同一代码，只换网关"，Phase 4 切实盘时信号层零改动。
5. 订单状态机参考 backtrader/vn.py 的 OMS 枚举（Submitted→Accepted→Partial→Completed/Canceled/Rejected），避免自己发明不完备的状态集。

### 复盘聚合（07-21 前自研，抄三家）

1. **延迟回填模式 → 抄 TradingAgents**：pending 信号在后续运行时用已实现收益 + 对沪深300 的 alpha 回填，并生成一段反思文字；
2. **决策日志与双基线 → 抄 AgenticTrading**（325 stars 小项目但正是我们要的最小蓝本）：每条决策记录 action/时间戳/价格/理由/组合快照；复盘页永远带两条基线（买入持有 + 沪深300 指数）；
3. **小样本统计诚实性 → 抄 PyBroker**：bootstrap 重采样置信区间——LLM 信号样本少，单点收益率会骗人，置信区间才诚实；
4. 加分项（来自 8 stars 微型项目 AlphaTrader 的点子）：滚动 holdout 给不同 prompt/模型的信号源打分赛马，做信号源淘汰机制——Phase 2 三大纪律 A/B 对比正好用。

### 筛选器（07-21 后自研，抄两家思路）

- vectorbt 的"信号布尔矩阵→一次算完全池历史表现"向量化思路（比逐股循环快两个数量级）；
- zvt 的 entity_id + timestamp 双索引 schema 组织横截面计算。

## 四、Phase 2 回测/因子选型短名单（07-21 复盘会后正式横评）

| 候选 | 定位 | 关键事实 |
|---|---|---|
| **RQAlpha** | 全仿真回测首选 | 6.5k stars，6.2.0 发布于 2026-06-25，持续活跃；原生 A 股规则；米筐公司背书 |
| **PyBroker** | 轻量信号驱动路线 | 原生支持 AKShare；止损止盈一等公民；自带 bootstrap 统计检验（对"LLM 信号有没有用"的检验特别有价值）；风险：单人维护 |
| **vectorbt 开源版** | 筛选器扫参专用 | 已复活（v1.1.0 发布于 2026-07-05，含 Rust 引擎）；Commons Clause 仅限售卖，自用无碍 |
| **Qlib + RD-Agent** | ML 因子路线（若走） | 微软背书，RD-Agent 活跃；等缩量池基线跑出来再决定是否上 ML |
| 观察名单 | Vibe-Trading | 2026-04 创建仅 3 个月已 18k stars，A 股数据工程最认真（数据源按封禁风险排序自动降级）、456 因子 Alpha Zoo；太新不做依赖，2026 年底复查 |

**时效性纠偏（与国内旧教程相反的四个事实）**：backtrader 已停更 3 年+（勿再跟旧教程）；WonderTrader 停滞 20 个月（勿被推荐文种草）；vectorbt 开源版已复活；TradingAgents-CN 29.8k stars 但已 2.5 个月停更 + 非标准许可证（双红旗，只当踩坑手册读）。

## 五、Phase 4 实盘通道现实边界（预研结论，离执行还很远）

1. **散户合规通道**（2026 现状）：券商版 QMT/miniQMT（迅投）与 Ptrade（恒生），门槛已下探至约 10-50 万（各券商差异大，落地前逐家确认，中等可信度）；需风险测评 C4+、6 个月交易经验。
2. **监管面**：程序化交易"先报告后交易"（2024-10 施行），高频认定阈值为 300 笔/秒或 2 万笔/日（2025-07 细则）——我们日线级策略差几个数量级，走券商通道天然合规；但"绕开券商系统外接直连"对散户已基本堵死，反向印证 miniQMT/Ptrade 路线。
3. **工具选型预埋**：个人户优先 easytrader/xtquant + miniQMT；vn.py 定位为"若上规模/上期货"的可选框架层。均符合"不自研券商接口"红线。

## 六、对三个自研件的最终确认

调研后结论不变且更有底气：执行器/筛选器/复盘聚合**维持自研**（胶水量级 + 全网无现成 + 借鉴地图已到文件级）；Phase 2 回测**必须复用**（短名单已定）；Phase 4 网关**必须复用**（路线已修正为 miniQMT 优先）。所有"借鉴"均抄设计不抄代码，不引入任何维护存疑的依赖——本领域 star 数泡沫严重（TradingAgents-CN 29.8k stars 与停更并存即是例证），star 不代表工程可靠性。
